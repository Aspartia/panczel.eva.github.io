HR Matcher Application

Embedding-based resume-to-role matching system with evidence retrieval and structured candidate summaries.

Python NLP Sentence Transformers FAISS TF-IDF SQL Server HR Analytics

Overview

This project focuses on building an intelligent HR support tool that compares resumes against job roles and identifies the most suitable candidates. The application reads resume data from a structured dataset, preprocesses the text, generates semantic embeddings, and ranks candidates based on their similarity to role descriptions.

The goal was not only to assign a likely role to each resume, but also to provide a more interpretable matching workflow through evidence snippets, experience estimation, and exportable candidate summaries.

Business Problem

In recruitment workflows, screening large numbers of resumes manually is slow and difficult to scale. A recruiter often needs to compare multiple candidates against multiple roles, while also understanding why a candidate was recommended.

High Resume Volume

Manual resume review becomes increasingly inefficient when many candidates and multiple positions must be evaluated in parallel.

Role Fit Ambiguity

Candidates often have overlapping skills, so matching them to the most suitable role requires deeper semantic comparison than keyword search alone.

Need for Explainability

Beyond ranking, HR users need interpretable output: role suggestions, evidence snippets, and an estimate of seniority or experience level.

What the Application Does

The HR Matcher application processes resumes as text documents, transforms them into embeddings, and compares them with job descriptions. Role descriptions can either be loaded from PDF-based job postings or replaced by predefined fallback roles.

Resume CSV → Text Cleaning → Chunking → Embedding Generation → FAISS / TF-IDF Retrieval → Role Matching → Experience Estimation → Summary Export

Input Data & Role Definitions

The system works with a resume dataset stored in CSV format and supports external role descriptions loaded from PDF files. This makes the workflow flexible: it can be adapted to real job postings instead of relying only on manually written text templates.

Resume Input

Resumes are read from a CSV file. The application searches for a resume-like text column and standardizes it for further processing.

PDF Role Loading

Job descriptions can be loaded directly from PDF files, allowing the system to work with realistic, externally prepared role documents.

Fallback Roles

If no PDFs are available, the application can still run using predefined example roles such as Data Scientist, SAP Developer, and Java Developer.

Job Role Examples

NLP & Retrieval Architecture

A core part of the project is the document retrieval logic. Resumes are cleaned, split into word-based chunks, and converted into dense vector representations using sentence embeddings. This makes it possible to compare semantically similar content even when exact keywords differ.

Text Cleaning

URLs, punctuation, noisy formatting, and non-standard characters are removed to make the resumes more consistent before vectorization.

Chunking

Long resumes are split into overlapping chunks, which improves evidence retrieval and makes the semantic search more precise.

Embeddings

SentenceTransformer models are used to embed both resumes and role descriptions into the same semantic vector space.

FAISS Search

FAISS is used to index vectorized resumes and support efficient similarity search for scalable role matching.

TF-IDF Fallback

If embedding-based retrieval is not available, the system can switch to a TF-IDF + cosine similarity fallback workflow.

Evidence Retrieval

The top chunks supporting a candidate-role match can be extracted, making the recommendation easier to interpret.

Matching Logic

The final candidate-role ranking is based on embedding similarity between resumes and job roles. For each resume, the application identifies the top predicted roles, while for each role it can also retrieve the most suitable candidate profiles.

Role Prediction

Each resume receives a ranked list of the most likely roles based on semantic similarity scores.

Top Candidate Ranking

For each job description, the application identifies the strongest candidate matches from the resume dataset.

Snippet-Level Evidence

The model can return the most relevant resume chunks to support why a certain candidate appears relevant to a specific role.

Data Science Results

Experience Estimation

In addition to role matching, the project includes a heuristic component that attempts to extract years of experience from resume text. Based on the detected values, candidates are assigned to a simplified experience category.

Years Detection

Regular-expression based extraction is used to detect mentions of years or months of experience in the resume text.

Bucket Assignment

  • Junior
  • Mid
  • Senior
  • Unknown

Practical Use

This extra feature helps make candidate summaries more useful in real HR review scenarios, especially when screening large applicant pools.

Output & Integration

The application produces a structured summary of candidates, including predicted top role, role scores, estimated experience, and a short resume snippet. The output can be exported as CSV and, optionally, written into SQL Server.

CSV Output

A timestamped CSV export contains the final candidate summary table, making the result easy to inspect and reuse.

SQL Server Save

When configured, the summary can also be stored in SQL Server for downstream reporting or further application use.

Metadata Logging

The run configuration and process metadata can also be stored to support reproducibility and structured project execution.

Key Takeaways

  • The project demonstrates practical NLP in a business context. It goes beyond text cleaning and uses embeddings for semantic matching.
  • It combines retrieval and scoring logic. The solution is not just a classifier, but a searchable recommendation workflow.
  • Interpretability was an important design goal. Evidence snippets and experience estimation make the results more useful for human reviewers.
  • The system is built in a modular way. This makes the project easier to maintain, extend, and integrate into broader HR analytics workflows.
