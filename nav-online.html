NAV Online Invoice Exporter

Python-based API integration for automated invoice retrieval, processing, and Excel export.

Python REST / XML API Automation Excel Export openpyxl Data Processing NAV Online Számla API

Overview

This project is an automated invoice retrieval and export solution built on top of the Hungarian NAV Online Számla API. It was designed to collect invoice digest data directly from NAV, process the returned XML responses, and export the final result into a structured Excel workbook.

The solution supports both OUTBOUND and INBOUND invoices, handles large date ranges through automatic slicing, and generates a reusable output that can support accounting, reporting, reconciliation, or further analytical workflows.

Business Problem

In many reporting and finance-related workflows, invoice data must be checked, extracted, and transformed manually from multiple systems. This is time-consuming, repetitive, and error-prone, especially when larger date intervals or multiple invoice directions must be processed.

Manual Workload

Downloading and filtering invoice data manually creates unnecessary administrative effort and slows down reporting cycles.

Technical Complexity

The NAV API requires XML request generation, signature creation, namespace handling, paging, and structured response parsing.

Reliable Export Need

Users often need a clean and directly usable Excel export for downstream processing, validation, or integration into BI workflows.

What the Solution Does

The script authenticates against the NAV Online Számla API using the required technical user credentials and request signature logic, then retrieves invoice digest data for a selected period. The response is parsed, selected fields are transformed into a tabular structure, and the results are saved into an Excel file.

Environment Variables → Signed XML Request → NAV API Query → Pagination Handling → Date Range Slicing → XML Parsing → Data Transformation → Excel Export

Core Technical Challenges

A key part of the project was not only accessing the API, but handling its constraints and business-specific requirements in a robust way.

Secure Request Signing

The API requires hashed authentication fields and a generated request signature based on timestamp and signing key.

35-Day Date Limit

Larger date intervals were automatically split into smaller slices to make the requests compatible with API constraints.

Pagination

The solution iterates through multiple result pages and stops automatically after reaching the last available page.

XML Parsing

The returned XML structure was processed with namespace-aware parsing to reliably extract the relevant invoice fields.

Inbound + Outbound Support

The workflow was extended to retrieve both invoice directions and store them in separate sheets and a merged summary output.

Excel Reporting Output

The final dataset is written into Excel to make the result directly usable in finance and reporting workflows.

Exported Data

The export contains multiple business-relevant invoice fields that can support checking, reconciliation, and reporting.

Invoice Identification

  • Invoice number
  • Original invoice number
  • Transaction ID
  • Modification index

Dates

  • Invoice issue date
  • Upload date
  • Payment due date
  • Delivery / fulfillment date

Amounts

  • Net amount
  • VAT amount
  • Gross amount
  • HUF values

Parties & Metadata

  • Supplier name
  • Customer name
  • Tax numbers
  • Payment method / invoice category

Output Structure

The generated Excel workbook is structured in a way that supports both detailed review and combined analysis.

OUTBOUND Sheet

Contains outgoing invoice records retrieved from the NAV system.

INBOUND Sheet

Contains incoming invoice records retrieved from the NAV system.

ALL Sheet

Merged and sorted view of both invoice directions for broader review and reporting.

Why This Project Matters

This project demonstrates practical software engineering in a business environment: API integration, data transformation, error handling, export automation, and workflow design. It is especially relevant for data analyst, BI, and data engineering roles, because it shows how raw operational data can be turned into a reusable and structured reporting asset.

  • API integration in practice: working with real-world external systems and technical authentication requirements.
  • Automation mindset: reducing repetitive manual work through reproducible scripting.
  • Business relevance: invoice data is directly useful in finance, reporting, reconciliation, and audit support processes.
  • Scalable logic: pagination, slicing, and structured export make the solution reusable across larger date ranges.
